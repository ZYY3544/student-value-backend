# 职场价值评估系统 - 技术说明文档

> 本文档面向开发人员，说明前后端各文件的功能和依赖关系，便于改造为微信小程序。

---

## 一、项目概述

这是一个「测一测身价」职场价值评估系统，用户上传简历(CV)或输入岗位职责(JD)，系统通过 AI + HAY 评估模型计算职级和薪酬范围。

### 核心评估流程

```
用户输入 (简历/JD)
    ↓
1. 文本解析 (支持 PDF)
    ↓
2. LLM 提取关键因素 (DeepSeek API)
    ↓
3. HAY 8因素评估 → 职级计算
    ↓
4. 8因素 → 8维能力映射 (雷达图)
    ↓
5. 薪酬查询 (城市/行业/职能)
    ↓
6. 生成趣味标签
    ↓
返回结果
```

#### 核心计算层

| 文件 | 功能 | 关键函数/类 |
|------|------|-------------|
| **incremental_convergence.py** | 增量收敛引擎，1次LLM调用生成30-80个方案 | `IncrementalConvergence` 类<br>`find_optimal_solution()` - 查找最优解 |
| **calculator.py** | HAY三维度八因素计算 | `HayCalculator` 类<br>`calculate_hay_evaluation()` - 返回总分、职级 |
| **profile_calculator.py** | 岗位特性计算 (P型/L型/A型) | `calculate_job_profile()` |
| **salary_calculator.py** | 薪酬分位值查询 (P25/P50/P75) | `SalaryCalculator` 类<br>`get_salary_range()` - 查询薪酬区间 |
| **ability_mapper.py** | 8因素→8维能力映射（1:1直接映射） | `map_factors_to_dimensions()` - 核心映射<br>`get_dimension_radar_data()` - 雷达图数据 |
| **level_tags.py** | 趣味标签生成 (搬砖圣手、六边形战士等) | `get_level_tag_and_desc()` |

**8维能力模型（1:1映射，避免HAY知识产权）：**
```
知识深度    ← PK  (专业知识的深度与广度)
统筹能力    ← MK  (组织协调与资源调度能力)
沟通影响    ← Comm (表达、说服与跨团队协作)
问题复杂度  ← TE  (面对问题的模糊程度与复杂性)
创新思维    ← TC  (突破常规、提出新方案的能力)
决策自主性  ← FTA (独立判断与自主决策的空间)
影响规模    ← M   (工作成果影响的范围大小)
贡献类型    ← NI  (对组织的贡献是辅助性还是主导性)
```

## 七、产品方向：能力定级 × 市场定薪（待实施）

> 2026-03-09 讨论确定，尚未改代码，等前端一起改。

### 问题

当前产品将「个人能力」和「市场定价」揉成一个薪酬数字输出。用户换个城市/行业，薪酬就变了——用户会困惑这个数字到底是评"我"还是评"市场"。

### 方案：拆成两层

**第一层：能力画像（关于你，不随城市/行业变化）**
- 趣味标签 + 描述（已有，替代裸露的职级数字）
- 8 维雷达图（已有）
