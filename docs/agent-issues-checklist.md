# Agent 对话行为问题清单

> 测试过程中发现的 Agent 回复问题 & 解决记录
> 最后更新：2026-03-05

---

**问题 1：用户点击蓝色气泡后，Agent 一上来就开始分析简历，用户还没提任何要求**
解决：重写 DiagnosisAgent prompt，开场白改为友好欢迎 + 功能介绍 + 问用户想从哪开始，严格不分析简历 ✅

**问题 2：Agent 回答中出现"HAY"、"八因素"、"岗位评估"等方法论术语**
解决：三个 Agent prompt 全部增加禁止提及规则，去除所有 HAY/KH/PS/ACC 等术语 ✅

**问题 3：Agent 把业务描述成"评价岗位"，但实际是评价人的能力和简历竞争力**
解决：prompt 中明确写入"我们评价的是人的能力和简历的竞争力，不是评价岗位" ✅

**问题 4：改写建议中直接体现八因素术语（如"影响范围(M)"），用户可能推导出方法论**
解决：能力维度锚定改为通俗语言，如"业务影响力"、"分析和解决问题的能力" ✅

**问题 5：用户追问评分逻辑时，不希望 Agent 说"不好意思不能告知"，但也不要透露底层**
解决：不拒绝，自然用通俗语言概括（如"我们从多个维度综合评估"），不透露公式和模型 ✅

**问题 6：语气太正式、太书面，希望模仿豆包风格**
解决：三个 Agent prompt 语气统一改为像靠谱学长/学姐帮忙改简历，口语化、鼓励性、轻松自然 ✅

**问题 7：用户问"你能帮我做什么"时，Agent 不仅回答了问题还直接开始改简历**
解决：OptimizeAgent prompt 增加对话节奏规则，问问题时简短回答，不改简历不举例 ✅

**问题 8：开场白需要更友好、更像欢迎语，介绍 Agent 功能**
解决：prompt 固定结构：欢迎句 + 已看过评测 + 三功能介绍（emoji + 加粗）+ 收尾提问 ✅

**问题 9：结果页的"深度分析结论"去掉，功能由聊天 Agent 承接**
解决：后端去掉 generate_deep_insight 调用和返回字段，前端去掉深度分析卡片和简历健康度环形图 ✅

**问题 10：用户直接问"HAY是什么？你们是不是用HAY"时，Agent 确认了使用 HAY**
解决：prompt 增加规则 — 不确认也不否认，自然说"我们有自己的评估方式"并转移话题回简历优化 ✅

**问题 11：改写建议中编造用户没提到的具体数字（如"GMV提升15%"），行为不稳定**
解决：prompt 强化红线 — 严禁编造数字，必须用「[待补充: 具体数字]」标记 ✅

**问题 12：用户只是打招呼（"你好"、"嗨"），Agent 直接开始分析简历**
解决：对话节奏规则改为 ABC 三类意图分级，打招呼和问问题严禁引用简历内容和评测数据 ✅
