## 四、Agent 对话行为

| # | 问题 | 状态 | 解决方案 |
|---|------|------|----------|
| 4.1 | 用户问"你能帮我做什么"时，Agent 不仅回答了问题还直接开始改简历 | ✅ 已完成 | OptimizeAgent prompt 增加【对话节奏】规则：先听清意图再行动，问问题时简短回答不改简历不举例 |
| 4.2 | Agent 回答中出现"HAY"、"八因素"等方法论术语 | ✅ 已完成 | 三个 Agent prompt 全部增加【绝对禁止提及的内容】规则，去除所有 HAY/KH/PS/ACC 等术语 |
| 4.3 | Agent 把我们的业务描述成"评价岗位"，但实际是评价人的能力和简历竞争力 | ✅ 已完成 | prompt 中明确写入："我们评价的是人的能力和简历的竞争力，不是评价岗位" |
| 4.4 | 改写建议中直接体现八因素术语（如"影响范围(M)"、"思维挑战(TC)"），用户可能推导出方法论 | ✅ 已完成 | 能力维度锚定改为通俗语言：如"业务影响力"、"分析和解决问题的能力"、"协作和沟通能力"等 |
| 4.5 | 用户追问评分逻辑时，不希望 Agent 说"不好意思不能告知"，但也不要透露底层 | ✅ 已完成 | prompt 规则：不拒绝，自然用通俗语言概括（如"我们从专业能力、业务影响、团队协作等多个维度综合评估"），不透露公式和模型 |
| 4.6 | 语气太正式、太书面，希望模仿豆包风格 | ✅ 已完成 | 三个 Agent prompt 语气统一改为：像靠谱学长/学姐帮忙改简历，口语化、鼓励性、轻松自然 |
