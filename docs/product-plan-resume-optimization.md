# 产品规划：简历优化功能方案

> 日期：2026-03-03
> 状态：内部讨论稿
> 背景：基于前期用户调研及高校教授反馈，探讨将"简历优化"作为核心付费功能的产品方案

---

## 一、问题背景

### 当前产品价值链

```
用户输入简历 → 薪资评估 → 深度分析 → ???
```

现有产品完成了"价值发现"和"价值解读"两个环节，但在"价值提升"环节断裂。学生看完分析后的核心疑问是：**"我知道我值这么多了，那我该怎么办？"**

### 调研结论

- 高校教授普遍认为：对学生来说，**最有价值的落脚点是改简历**
- 逻辑链：知道值多少 → 理解为什么 → 知道如何提升 → **实际改善简历**
- 当前深度分析中的改善建议偏泛泛，缺乏针对性和可操作性

### 核心判断

不是做一个独立的"AI 改简历工具"（市面上已经很多），而是利用我们独有的 **HAY 评估体系**，做一个**基于岗位价值评估的简历优化**——这是别人做不了的差异化。

---

## 二、产品架构：三层递进模型

### 整体设计

```
┌─────────────────────────────────────────────────────────────┐
│  第一层：价值发现（免费 / 低价引流）                          │
│  核心问题："你值多少钱"                                      │
│  ───────────────────────────────────────────                  │
│  输出：薪资区间 + 趣味标签 + 五维雷达图 + 简历健康分          │
│  目的：制造认知冲击，驱动分享传播                              │
│                                                              │
│  用户心理：哇，原来我值这么多 / 这么少                        │
└──────────────────────────┬──────────────────────────────────┘
                           │ "为什么？我要看看"
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  第二层：价值解读（当前深度分析，可作为付费门槛）              │
│  核心问题："为什么值这么多"                                   │
│  ───────────────────────────────────────────                  │
│  输出：深度洞察（4 段分析）+ 核心竞争力 + 面试要点            │
│  目的：建立信任感，让用户认可评估的专业性                      │
│                                                              │
│  用户心理：分析得很准，那我怎么才能更值钱？                    │
└──────────────────────────┬──────────────────────────────────┘
                           │ "帮我改简历吧"
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  第三层：价值提升（核心付费功能，高客单价）     ⭐ 新增功能    │
│  核心问题："怎么让你更值钱"                                   │
│  ───────────────────────────────────────────                  │
│  输出：简历诊断报告 + 逐条优化建议 + 改写示范 + 预估提升      │
│  目的：真正帮学生解决问题，产生付费意愿                        │
│                                                              │
│  用户心理：值了，改完简历确实更好了                            │
└─────────────────────────────────────────────────────────────┘
```

### 三层之间的转化逻辑

| 层级 | 用户状态 | 转化驱动力 |
|------|---------|-----------|
| 第一层 → 第二层 | 好奇心被激发 | "为什么我值这个价？" |
| 第二层 → 第三层 | 信任感建立 + 提升欲望 | "分析很准，帮我改简历能多值多少？" |

## 三、第三层功能详细设计（简历优化）

### 模块 1：简历诊断报告（基于 HAY 因素的独特诊断）

**差异化价值**：这不是市面上那种"你的简历缺少量化数据"的通用诊断，而是基于 HAY 8 因素评估结果的**针对性诊断**。

**产品形态**：

```
你目前的评估结果：
┌──────────────────────────────────────────────────────┐
│  专业力 (PK): D+    ← 你的简历未能充分体现            │
│  思辨力 (TE): C     ← 表现良好 ✓                     │
│  管理力 (MK): T     ← 有提升空间                     │
│  创新力 (TC): 3     ← 表现良好 ✓                     │
│  合作力 (Comm): 2   ← 需要补充                       │
└──────────────────────────────────────────────────────┘

诊断详情：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📋 专业力（最大提升空间）
你的简历在"专业力"维度只体现出了 D+ 水平，但根据你的实际
经历（XX 项目、XX 实习），合理水平应在 E 档。主要原因是：
你的项目描述缺乏技术深度的展现，且未量化具体成果...

📋 合作力（建议补充）
你的简历中缺少团队协作相关的描述。建议在项目经历中补充
你在团队中的角色、跨部门协作经验以及沟通协调的具体案例...
```

**为什么只有我们能做**：因为我们有 HAY 评估体系，能精确指出"简历的哪个维度拖了后腿"，而不是给出通用的模板化建议。

---

### 模块 2：逐条经历优化（Before → After）

**核心逻辑**：针对简历中的每一段经历 / 项目，给出具体的改写建议，并关联到 HAY 因素的变化。

**产品形态**：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【实习经历 #1】

原文：
  "在 XX 公司实习，负责数据分析工作，完成了多个分析报告"

问题诊断：
  ❌ 缺少量化成果（影响 专业力 评分）
  ❌ 未体现独立思考（影响 思辨力 评分）
  ❌ 职责描述模糊，无法判断工作深度

优化建议：
  ✅ "在 XX 公司数据分析部门实习期间，独立搭建用户留存
     分析模型，覆盖 3 个核心业务线、日均处理 10 万+ 数据量，
     产出的分析报告直接推动产品优化，用户 7 日留存率提升
     2.3 个百分点"

预估影响：专业力 D+ → E-（+0.5 档）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【项目经历 #2】

原文：
  "参与了校内创新创业比赛，获得三等奖"

问题诊断：
  ❌ 未说明具体做了什么（影响 专业力 评分）
  ❌ 缺少团队角色描述（影响 管理力 / 合作力 评分）
  ❌ 成果描述单薄

优化建议：
  ✅ "作为 4 人团队的技术负责人，主导开发 XX 智能推荐系统，
     负责算法设计与模型调优，最终方案在校内创新创业大赛中
     从 120 支队伍中脱颖而出获得三等奖"

预估影响：管理力 T → T+（+0.5 档），合作力 2 → 2（不变，但信息更完整）
```

**核心差异化**：每条建议都关联到具体的 HAY 因素提升，不是泛泛的"写得更好"。

**核心逻辑**：基于用户选择的目标岗位职能（当前已有 34 个职能分类），给出岗位适配度分析和针对性建议。

**产品形态**：

```

🎯 目标岗位匹配分析

该岗位最看重的 TOP 3 能力维度：
┌────────────────────────────────────────────────┐

关键差距：专业力（差 1 档）

## 四、定价策略

### 方案 A（推荐）：两级定价

| 版本 | 价格 | 包含内容 |
|------|------|---------|
| **基础版** | 9.9 ~ 19.9 元 | 第一层（薪资评估 + 标签 + 雷达图）+ 第二层（深度分析） |
| **进阶版** | 49.9 ~ 99 元 | 基础版全部 + 第三层（简历诊断 + 逐条优化 + 岗位适配 + 价值提升预估） |

**推荐理由**：
- 基础版价格低，降低首次付费门槛
- 学生看完基础版分析后，自然产生"帮我改简历"的需求
- 进阶版的价格锚定在"一杯奶茶 vs 可能多挣几万块"，决策成本极低
- 转化链路清晰，不需要过多教育用户

### 方案 B：三级定价

| 版本 | 价格 | 包含内容 |
|------|------|---------|
| **体验版** | 免费 | 第一层（薪资 + 标签 + 雷达图） |
| **标准版** | 19.9 元 | + 第二层（深度分析） |
| **旗舰版** | 69.9 ~ 99 元 | + 第三层（简历优化全套） |

**适用场景**：如果希望最大化传播量，免费体验层可以快速起量。

### 方案 C：单一定价

| 版本 | 价格 | 包含内容 |
|------|------|---------|
| **全套版** | 39.9 ~ 59.9 元 | 三层全部功能 |

**适用场景**：简化运营，适合早期快速验证。

## 五、技术可行性评估

### 现有技术资产可直接复用

| 能力 | 现有基础 | 复用方式 |
|------|---------|---------|
| 简历解析 | PDF / DOCX / 纯文本解析已完成 | 直接复用 `parse_pdf_from_base64()` 等 |
| HAY 评估 | 完整的 8 因素 → 5 能力映射 | 评估结果直接作为诊断依据 |
| 岗位数据 | 34 个职能分类 + profile 数据 | 用于目标岗位适配分析 |
| 薪资引擎 | 完整的薪资计算链路 | 可模拟优化后的薪资预估 |
| LLM 集成 | DeepSeek 调用链路成熟 | 新增 prompt 模块即可 |

### 需要新增的技术模块

| 模块 | 复杂度 | 说明 |
|------|--------|------|
| 简历结构化拆分 | 中 | 将简历拆分为教育经历、实习经历、项目经历等段落 |
| 逐条诊断 prompt | 中 | 基于 HAY 因素，对每段经历生成针对性诊断 |
| 改写建议生成 prompt | 中 | 生成 Before → After 改写示范 |
| 价值提升模拟计算 | 低 | 基于优化后的预估因素，重新走一遍薪资计算链路 |
| 岗位能力权重配置 | 低 | 为 34 个职能配置 TOP 能力维度及目标值 |

### 预估开发周期

- **MVP 版本**（模块 1 + 模块 4）：可快速验证核心价值
- **完整版本**（4 个模块全部）：在 MVP 验证后迭代

## 六、MVP 建议

如果要快速验证，建议先做 **模块 1（简历诊断报告）+ 模块 4（价值提升预估）**：

- **模块 1** 让用户看到"我的简历哪里有问题"——制造痛点
- **模块 4** 让用户看到"改完能多挣多少钱"——提供动力
- 这两个模块技术实现相对简单，且足以验证用户付费意愿

模块 2（逐条优化）和模块 3（岗位适配）可在 MVP 验证通过后迭代上线。

## 七、竞争壁垒分析

| 维度 | 通用 AI 改简历工具 | 我们的方案 |
|------|-------------------|-----------|
| 诊断依据 | 通用规则（量化、动词等） | HAY 8 因素评估结果 |
| 优化方向 | 泛泛的"写更好" | 精确到每个能力维度的提升 |
| 价值量化 | 无 | 优化后薪资预估（有数据支撑） |
| 岗位针对性 | 弱 | 基于 34 个职能分类的精准匹配 |
| 用户信任 | 低（不知道凭什么改） | 高（先通过评估建立专业信任） |

**核心壁垒**：HAY 评估体系 + 薪资数据库 + 岗位职能数据，三者结合形成的**评估-诊断-优化闭环**，是通用 AI 工具无法复制的。

## 八、简历优化 Agent（Chatbot）实现方案

> 决策：第三层"价值提升"功能以 **对话式 Agent（Chatbot）** 形态实现，嵌入网页端。

### 8.1 为什么用 Chatbot 而不是一次性报告

| 维度 | 一次性报告 | Chatbot Agent |
|------|-----------|---------------|
| 信息获取 | 只能基于简历已有内容 | 可主动追问，挖掘简历未写但实际做过的事 |
| 用户体验 | 冷冰冰的报告 | 像和职业导师对话，有温度 |
| 优化精度 | 泛泛建议 | 多轮迭代，每条建议都可细化 |
| 信息不足问题 | 无解 | 通过对话自然补充 |
| 付费感知 | "看了一页报告" | "和 AI 导师聊了 20 分钟"，价值感更强 |

### 8.2 Agent 对话流程设计

```

│                                                                  │

│  • 原始简历文本                                                  │
│  • HAY 8 因素评估结果                                            │
│  • 5 维能力得分                                                  │
│  • 薪资区间 + Grade                                              │
│  • 深度洞察文本                                                  │
│  • 简历健康分                                                    │
│  • 目标岗位信息（城市/行业/职能/职位）                            │
│  • 学校/学历信息                                                 │
└──────────────────────────┬──────────────────────────────────────┘
                           │ 点击"优化我的简历"
                           ▼
┌─────────────────────────────────────────────────────────────────┐
│  Phase 0：开场白（Agent 主动发起）                                │
│  ─────────────────────────────────────                           │
│                                                                  │
│  Agent：                                                         │
│  "你好！我已经看过你的简历和评估结果了。你目前的评估是              │
│   Grade {X}，月薪 {range}。整体来看你的简历健康分是 {score}，     │
│   主要在{最弱维度}方面还有不少提升空间。                           │
│                                                                  │
│   接下来我会逐条帮你优化简历中的经历描述，每一条我都会              │
│   告诉你改动对应的价值提升。我们开始吧！                          │
│                                                                  │
│   先来看你的第一段经历——"                                        │
└──────────────────────────┬──────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────────┐
│  Phase 1：逐条经历优化（核心循环，每段经历重复此流程）            │
│  ─────────────────────────────────────                           │
│                                                                  │
│  Step 1 - 诊断                                                   │
│  Agent：                                                         │

#### 流式输出实现（SSE）

```python
# 当前 LLM 调用方式（同步，需改造）
# llm_service.py 中新增流式方法

def stream_chat_completion(self, messages, on_chunk=None):
    """流式调用 DeepSeek，逐 token 返回"""
    response = self.client.chat.completions.create(
        model=self.model,
        messages=messages,
        temperature=0.7,      # 对话场景适当提高温度
        stream=True           # 关键：开启流式
    )
    full_content = ""
    for chunk in response:
        if chunk.choices[0].delta.content:
            token = chunk.choices[0].delta.content
            full_content += token
            if on_chunk:
                on_chunk(token)
            yield token
    return full_content
```

### 8.7 成本预估

```
单次完整对话的 LLM 调用成本（DeepSeek）：

• 简历结构化拆分：1 次调用 ≈ 2k tokens
• Agent 对话：约 10-20 轮 × 平均 2k tokens/轮 ≈ 20k-40k tokens
• System Prompt（含上下文）：每轮约 1.5k tokens
• 总计：约 35k-60k tokens / 次完整对话

DeepSeek 定价（参考）：
• 输入：¥1 / 百万 tokens
• 输出：¥2 / 百万 tokens
• 单次对话成本约：¥0.05 - ¥0.10

结论：LLM 成本极低，即使定价 9.9 元也有足够利润空间。
```

### 8.8 关键风险 & 应对

| 风险 | 影响 | 应对策略 |
|------|------|---------|
| Agent 回复质量不稳定 | 用户体验差 | Phase 1 收集真实对话数据，迭代 Prompt；加入人工审核采样 |
| 对话过长用户失去耐心 | 完成率低 | 控制单次回复长度 ≤200 字；允许跳过不想优化的段落 |
| 用户期望过高 | 口碑风险 | 开场白明确定位"建议"而非"保证"；价值预估标注"仅供参考" |
| LLM 编造用户没有的经历 | 信任崩塌 | Prompt 严格约束：只能基于用户提供的信息改写，不能凭空编造 |
| 并发压力（多人同时对话） | 服务不稳 | Session Manager 支持多会话隔离；DeepSeek API 有速率限制需处理 |
